## 1. Назначение и цели

**Продукт:** CRM-система «Мой Парк» — веб-приложение для управления таксопарком.

**Заказчик:** ИЖАВТОХОЛДИНГ (Удмуртская Республика)

**Регион:** Ижевск и Удмуртская Республика
**Размер парка:** 50–500+ автомобилей
**Модель работы:** преимущественно аренда ТС
**Агрегаторы:** Яндекс Go (основной), Такси 434343, Такси 373

**Цели системы:**
- Автоматизация управления водителями, автопарком и финансами
- Сверка и импорт данных из Яндекс Go
- Контроль документов (права, страховки, ТО) с уведомлениями
- Управление сменами и выручкой
- Финансовая аналитика и отчётность
- Управление техобслуживанием и расходами на ремонт

**Что НЕ входит в MVP (отложено на следующий этап):**
- SaaS / мультитенантность (несколько парков)
- Мобильное приложение водителя
- Интеграция с Госуслугами / ЕСИА
- 1C бухгалтерия
- Интеграция с GPS/телематикой

## 2. Технологический стек

### Frontend (основной)

| Технология | Версия | Назначение |
|------------|--------|------------|
| Vue 3 | ^3.5 | UI-фреймворк (Composition API) |
| PrimeVue 4 | ^4.5 | Компонентная библиотека |
| Pinia | ^3.0 | Стейт-менеджмент |
| Vue Router | ^5.0 | Роутинг |
| Tailwind CSS | ^4.2 | Утилитарные стили |
| PrimeIcons | ^7.0 | Иконки |
| Vite | ^8.0 | Сборщик |
| Axios | ^1.x | HTTP-клиент |

### Backend-as-a-Service

| Технология | Назначение |
| ------------ | ------------ |
| **Integram V2** | Основная база данных, REST API, файловое хранилище |
| Integram база: `kval` | Хранение всех данных CRM |

### Инфраструктура

| Компонент | Решение |
| ------------ | ------------ |
| Сервер | Contabo VPS (Linux) |
| Контейнеризация | Docker + Docker Compose |
| Веб-сервер | Nginx |
| Деплой | GitHub → pull + docker compose up |

### Почему Integram вместо Laravel + PostgreSQL

Оригинальное ТЗ предполагало Laravel + PostgreSQL. Адаптация на Integram V2 обоснована:

- Integram предоставляет полный REST CRUD API из коробки — отдельный backend не нужен
- Все таблицы, поля и связи настраиваются в UI Integram без написания кода
- Загрузка файлов, поиск, фильтрация — встроены в Integram API
- Сокращение слоёв: только Frontend + Integram API
- Уже развёрнут и настроен на сервере

## 6. Авторизация

### Схема входа

```
/login → Email + Пароль
       → bcrypt.compare(password, hash из Transport_Пользователи)
       → Генерация JWT → localStorage
       → Redirect на /dashboard
```

### Реализация без backend

- Пользователи хранятся в таблице `Transport_Пользователи` в Integram
- Хэш пароля: `bcryptjs` (npm пакет, работает в браузере)
- Токен: JWT, подписанный секретом из `.env` (`VITE_JWT_SECRET`)
- Сессия: токен хранится в `localStorage`, expires 24ч

> **Важно для production:** добавить Nginx-прокси или Cloudflare Worker как auth-шлюз,
> чтобы не хранить JWT_SECRET на клиенте. В MVP — допустимо для внутреннего инструмента.

### Защита маршрутов

```js
// router/index.js
router.beforeEach((to) => {
  const auth = useAuthStore()
  if (to.meta.requiresAuth && !auth.isLoggedIn) {
    return '/login'
  }
})
```

**Таблица (PrimeVue DataTable):**
- Колонки: Гос. номер · Марка/Модель · Год · Статус · Водитель · ОСАГО до · ТО следующее · Пробег
- Сортировка по всем колонкам
- Фильтры: статус, тип, водитель, истекающие документы
- Поиск: по гос. номеру, марке, VIN
- Пагинация: 20/50/100

**Цветовые статусы:**

| Статус | Цвет |
| ------------ | ------------ |
| Свободен | Зелёный |
| На смене | Синий |
| В ремонте | Жёлтый |
| Снят | Серый |

### Карточка автомобиля (URL: `/vehicles/:id`)

Табы:
1. **Детали** — все поля авто + форма редактирования
2. **Документы** — ОСАГО, КАСКО, ПТС (файлы + даты + напоминания)
3. **ТО и ремонт** — история из Transport_ТО
4. **Водители** — история привязанных водителей
5. **Финансы** — расходы на этот автомобиль из Transport_Финансы
6. **Штрафы** — штрафы по этому авто из Transport_Штрафы

### Форма добавления/редактирования

**Обязательные поля:** Гос. номер, Марка, Модель, VIN, Статус
**Валидация:** формат гос. номера (А123БВ 45 или А123БВ45), VIN (17 символов)

### Карточка водителя (URL: `/drivers/:id`)

Табы:
1. **Профиль** — личные данные, статус, тип занятости
2. **Документы** — ВУ, паспорт, СНИЛС (файлы + даты срока действия)
3. **Смены** — история смен из Transport_Смены
4. **Финансы** — выплаты, долги, операции из Transport_Финансы
5. **Автомобили** — история привязанных авто
6. **Штрафы** — штрафы из Transport_Штрафы
7. **Договора** — договора из Transport_Договора

### Раздел «Должники» (URL: `/drivers/debtors`)
- Таблица водителей с `Долг > 0` или `Баланс < 0`
- Быстрые действия: «Создать выплату», «Списать долг»

### Уведомления о документах
- ВУ истекает < 30 дней → жёлтый бейдж
- ВУ истекло → красный бейдж + предупреждение

## 10. Модуль: Смены

### URL: `/shifts`

### Список смен

**Таблица:**
- Колонки: Дата · Водитель · Авто · Статус · Выручка итого · Заказов · К выплате
- Фильтры: статус, водитель, авто, период
- Итоговая строка: сумма выручки за период

### Открытие смены

1. Выбрать водителя (поиск по ФИО)
2. Выбрать автомобиль (только со статусом «Свободен»)
3. Установить % комиссии парка
4. «Открыть смену» → статус авто меняется на «На смене»

### Закрытие смены

- Ввести выручку Яндекс и выручку наличными
- Ввести пробег и кол-во заказов
- Система считает: `К_выплате = Выручка_итого × (1 - Комиссия_парка / 100)`
- «Закрыть смену» → статус авто «Свободен»

### Импорт из Яндекс Go

- Кнопка «Синхронизировать с Яндекс»
- Выбор периода
- Загрузка транзакций водителей, автоматическое заполнение выручки

Решение: Nginx reverse proxy на сервере:
```nginx
location /yandex-api/ {
    proxy_pass https://fleet-api.taxi.yandex.net/;
    proxy_set_header Authorization "Bearer $YANDEX_KEY";
}
```
