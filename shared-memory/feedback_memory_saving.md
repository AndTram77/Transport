Сохранять важные решения в memory/ СРАЗУ в момент принятия, не откладывая на конец сессии.

**Why:** В сессиях 1–4 ни одно решение не было записано в память. К сессии 5 память была пустой — потеряли: решение по деплою, стек, ID таблиц Integram, GitHub репо, информацию о заказчике. Андрей вынужден был восстанавливать контекст устно.

**How to apply:** Как только в разговоре принято решение (сервер, порт, таблица, библиотека, договорённость) — сразу создать/обновить файл в memory/ и обновить MEMORY.md. Не ждать конца сессии. Перед финальным коммитом — проверить: всё ли записано.
