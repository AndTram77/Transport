# Дизайн схемы данных — Transport CRM "Мой Парк"

**Дата:** 2026-04-21
**Workspace:** Integram `taxi` (ai2o.ru/taxi/)
**Подход:** Плоская структура — все таблицы на одном уровне, связи через ref-колонки

---

## Обзор

CRM для управления таксопарком (50–500+ авто), ИЖАВТОХОЛДИНГ, Удмуртия.

- **21 таблица:** 14 справочников + 7 основных
- **~75 колонок** суммарно
- **5 ролей:** Админ, Диспетчер, Механик, Бухгалтер, Водитель
- **Задел:** интеграция с Яндекс Такси (через поле "Источник заказа")

## 3. Связи между таблицами

```
Пользователи ──→ Роли
Водители ──→ Пол, Статусы водителей, Автомобили
Автомобили ──→ Марки авто, Типы кузова, Типы топлива, Статусы авто
Смены ──→ Водители, Автомобили, Статусы смен
Рейсы ──→ Смены, Водители, Автомобили, Способы оплаты, Источники заказа, Статусы рейсов
Финансы ──→ Категории финансов, Типы финоперации, Водители, Автомобили
ТО ──→ Автомобили, Типы работ ТО
```

## 5. Задел на интеграцию с Яндекс Такси

- Рейсы: поле `Источник → "Яндекс"` маркирует рейсы от API
- Водители: в будущем добавить поле `Яндекс ID`
- Финансы: тип операции "Комиссия" будет включать комиссию Яндекса
- Отдельную таблицу НЕ создаём — достаточно ref на справочник "Источники заказа"
- API credentials парка (Clid, API-ключ) хранятся в конфигурации приложения, не в БД

## 7. Что делать с существующими таблицами в taxi

Текущие таблицы (Пользователь, Роль, Пол, Автомобиль, Марка, Водитель, Рейтинг, Клиент, Рейс, Состав работ) — черновой набросок без данных. Удалить после создания новой схемы, либо оставить в категории "Скрытые" на время миграции.
